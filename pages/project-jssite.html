Frog Pond Website!

This is my take on our Javascript Webshop project!


Bit of history:

Somewhere last year a desktop pet game called “Frog Pond” was released, where you can collect different frogs by attracting them with various decorations. Immediately I wanted to create a modification for the game, adding my own decorations and a new frog. Unfortunately that idea was quickly turned down since I couldn't access the source code. Luckily soon this school project feel into my hands.


Styling choices and End Goal:

The plan was to create a webshop with just four items including their prices and stock amount, which could be added and removed to/from the cart. And after a purchase, the user's money (a.k.a ShinyStones) would be affected.



Since I was now making the webshop directly inspired by the game, the colors followed along. I also tried to copy the creator's art style for the products.



With purchasing the items all either together or separately, the user would get a popup that a new frog wants to visit’s users pond, after closing which would show a new button that leads user to a new page where they can see the pond (design of which I directly took from the game) along with freshy bought decorations and a new visitor :3


Conclutions and Learned skills:

I used Javascript for the whole shop system which allowed me to add and substract items in the cart, as well as calculate the amount of money the user has left and display it all. It’s also the first website I’ve added a Favicon to.

Unfortunately I didn’t know about responsiveness so this website only looks good at 16x9 ratio.

During the process of building it I often looked at Shar’s (the creator of the game) portfolio website and I learned to use the debug tool to look into the code and actually picked up a few things I used in my project, but mainly it can be seen with box-shadow, which I haven’t used prior.

Overall I’m extremely satisfied with how it turned out, and I think it might've started my interest towards building shop systems.
